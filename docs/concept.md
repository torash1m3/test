# NeoForge — Концепция проекта

## Идея

**NeoForge** — платформа для энтузиастов сборки ПК. Не просто калькулятор, а полноценный хаб: собираешь конфигурацию, отслеживаешь покупки, общаешься с комьюнити и показываешь свой сетап.

---

## Ключевые фичи (из головы пользователя)

### 1. Конструктор ПК (Builder)
- Ручной ввод комплектующих (без встроенной БД компонентов)
- Drag & Drop для перетаскивания компонентов между категориями
- Калькулятор общей стоимости (авто-подсчёт)

### 2. Трекер покупок
- Статус каждой комплектующей: «Планирую» → «Куплено» → «Доставлено»
- Визуальный прогресс-бар всей сборки

### 3. Авторизация
- Логин / Регистрация
- Профили пользователей

### 4. Профили
- Аватарка + никнейм
- Описание своего текущего ПК и его характеристик
- Публичная видимость (все пользователи видят профили друг друга)

### 5. Форум
- Раздел «Новости» — посты с обсуждениями
- Раздел «Чат» — общение в реальном времени

### 6. Дизайн
- Тёмные оттенки
- Плавные анимации
- Передовой, премиальный вид

## Рекомендации

### Архитектура
- **Feature-Sliced Design (FSD)** — каждый модуль (auth, builder, forum, profile) живёт в своей папке со своими компонентами, стилями и стором. Это критично для масштабирования: хочешь добавить новый модуль — просто создаёшь новую папку, не трогая остальное.
- **Zustand вместо Redux** — для проекта такого масштаба Redux избыточен. Zustand даёт те же возможности (персистенция, мидлвары, devtools) без тонн бойлерплейта.
- **Lazy loading страниц** — каждая страница грузится по требованию через `React.lazy()`. Это напрямую влияет на скорость первой загрузки.

### Дизайн
- **Glassmorphism + subtle gradients** — полупрозрачные карточки с blur-эффектом на тёмном фоне. Выглядит дорого, реализуется просто.
- **Micro-animations** — hover states, появление элементов, плавные переходы между страницами. Framer Motion позволяет это делать декларативно.
- **Цветовая палитра**: основа — глубокий тёмно-синий/графитовый, акцент — неоновый cyan/violet. Не кислотно, но ярко на фоне тёмной темы.

### Данные
- На первом этапе — **всё в localStorage** через Zustand persist middleware. Это позволяет разрабатывать весь фронт без бэкенда.
- Когда понадобится бэкенд — Supabase (PostgreSQL + Auth + Realtime) будет идеальным выбором: бесплатный тир, встроенная авторизация, real-time подписки для чата.

## 5 предложений от меня

### 1. 🔌 Система совместимости комплектующих
Даже без встроенной БД можно дать пользователю выбирать *тип сокета*, *форм-фактор* и показывать предупреждения о несовместимости. Например: «AM5 процессор не встанет в LGA1700 мат. плату». Это превращает калькулятор из блокнота в умный инструмент.

### 2. 📊 Экспорт/Импорт сборки (JSON + красивый PDF)
Возможность сохранить сборку в JSON (для шаринга/бэкапа) и экспортировать в PDF с красивой вёрсткой. Это делает платформу полезной даже без аккаунта — просто собрал, скачал, отправил другу.

### 3. 🏆 Система достижений / бейджей
Бейджи в профиле: «Первая сборка», «Бюджетный мастер (сборка < 50к)», «Энтузиаст (сборка > 200к)», «Активный на форуме». Геймификация мотивирует пользователей возвращаться и участвовать в комьюнити.

### 4. 🔄 Сравнение сборок
Возможность положить две сборки рядом и сравнить: общую цену, количество купленных компонентов, прогресс. Полезно когда выбираешь между двумя конфигурациями.

### 5. 🌐 Публичная галерея сборок
Главная оживает когда на ней видны реальные сборки других пользователей — с ценами, статусами, комментариями. Это создаёт ощущение живого комьюнити и даёт новым пользователям вдохновение.
